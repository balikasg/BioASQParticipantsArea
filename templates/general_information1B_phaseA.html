Task 1B-Phase A Guidelines

For a comprehensive description of the BioASQ tasks please visit bioasq.org.
Phase A of Task 1B will begin on Monday 3rd of June, 2013. Each week, a new test set will be released. Participants, after downloading the test set will have to respond in limited time with the annotations for the questions of the test set that their systems estimated.

Training Data

The training data for both phases of Task 1B can be downloaded from Submitting/Task 1B-Phase A. The training set consists of 30 annotated questions in English. The questions have been created from biomedical experts around Europe. Four types of questions will be available:

  1. Yes/no questions (decisive)
  2. Factoid questions
  3. List questions
  4. Summary questions
The annotations for each question include lists of:
  • concepts from the designated ontologies and terminologies
  • relevant articles to the question from MEDLINE
  • snippets from the articles
  • RDF triples
Finally, an exact and an ideal answer that will be used in Phase B of Task 1B is also included.

Ontologies

The ontology sets that are available for Task 1B are:
  • the Medical Subject Headings (MeSH)
  • the Gene Ontology (GO)
  • the The Universal Protein Resource (UniProt)
  • the Joint Chemical Dictionary (Jochem)
  • the Disease Ontology (DO)
The above mentioned sets of ontologies are available for downloading in Submitting/Task 1B-Phase A.

Data Triples

The annotations of this phase of the task requires the usage of biomedical data expressed as triples, e.g., subject- predicate-object structured facts, extracted from biomedical resources or bibliography. The Linked Life Data project provides the LinkedLifeData platform. LinkedLifeData is a data warehouse that syndicates large volumes of heterogeneous biomedical knowledge in a common data model. The platform uses an extension of the RDF model that is able to track the provenance of each individual fact in the repository and thus update the information. It contains currently more than 8 billion statements, with almost 2 billion entities involved. The statements are extracted from 26 biomedical resources, such as PubMed, UMLS, DrugBank, Diseasome, and Gene Ontology. [Todo expand more the way they will search them and submit them to us]

Test set

Each test set will consist of non annotated questions in a JSON string. The format of the JSON string that will contain the questions will be the following: 
	{[
		{"questionId": "0001",
		 "questionType":"decisive" 			
		 "questionBody": "Do CpG Islands colocalize with 
		 transcription start sites"},
			    .
			    .      ]}
where:
  • questionId: is the unique identifier of each question
  • questionBody: is the question as it was formulated by the biomedical experts
  • questionType: is the type of the question with values:
    • "desicive", for yes/no questions.
    • "factoid", for factoid questions.
    • "list", for list questions.
    • "summary", for summary questions.
The JSON string represents an array with question objects. Each object has a questionId, a questionType and a questionBody. The questionId is the unique identifier of each question and will also be used during submitting results. Only registered users can download the test sets with one of the following ways:
  1. Using the web interface. In Submitting/Task 1B-Phase A you can find the available test sets. By clicking in the test (i.e. Test 1) you can download it as a text file. The test file will contain the json string with the questions.
  2. Using an API. By making a GET request in the URi http://bioasq.lip6.fr/tests/1BA/test_number/ along with authentication parameters. In the position of test_number there should be a number indicating the test set that the user wants to download. In the following section "Code Snippets" you can find snippets in Python that perform this request.

Submit Test Results

Participants of Phase A of task 1B will have to respond with lists of:
  • relevant concepts,
  • MEDLINE documents,
  • snippets from the documents and
  • RDF Triples
for each question. The concepts will come from the ontologies described above. The documets will be articles published in MEDLINE until 14th of March, 2013. The snippets will be parts of those articles. Finally, the RDF triples will come from the Linked Life Data platform.

Systems

Each user will have the opportunity to participate in Task 1B with a maximum of 5 systems. To enable this, after logging in you have to go to "Edit Profile Settings" and then click on the "Add System" button. The system name you will fill in the form will be the identifier of your system and it will be used in the "Results" section.

ATTENTION: Trying to upload results without selecting a system will result in error while the results will not be saved.

Cluster

A large computer cluster has been made available to BioASQ. Users can run their code to the cluster for both Tasks of BioASQ Challenge. Information and details about the procedure can be found in Submitting/Task 1A section. For more specific details you can also post in the BioASQ Discussion Forum.
